<!-- sep -->

## モチベーション

Windows機とLinux機でコードの共有を行う. 少なくともどちらからも開発可能にしておく.

## Git for Windows

[公式](https://gitforwindows.org/)からダウンロードすれば使えるが, いくつか注意点がある.

### Configuring the ending conversions

いわゆるLFかCRLFかの違いである.

Checkout as-is, commit Unix-style line endings

という2番目の選択肢を選ぶと良いらしい. 以下のコマンドでも変更できる.

```bash
# git config --global core.autocrlf true
git config --global core.autocrlf input
# git config --global core.autocrlf false
```

[気をつけて！Git for Windowsにおける改行コード](https://qiita.com/uggds/items/00a1974ec4f115616580)

[Configuring Git to handle line endings](https://docs.github.com/en/github/getting-started-with-github/getting-started-with-git/configuring-git-to-handle-line-endings)

## GitHubへのSSH接続

GitHub Desktopを使うと認可だけでSSH接続は不要です. ただしプロジェクトによってはWSL2とかを使ったほうがいいかもしれません. Node.jsとかRsutとかそんな感じのものを動かす場合です. この場合はLinuxのやり方に準拠するだけですが.
