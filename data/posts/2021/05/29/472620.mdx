## ランチャーからの起動

このままではコマンドラインからしか起動できないので不便です. ビルド後のフォルダにGNOMEのデスクトップ・エントリファイルがあるのでそれを利用します.

```bash
$ find ./ | grep -e ".desktop$"
./Engine/Source/Programs/UnrealVersionSelector/Private/Linux/Resources/com.epicgames.UnrealVersionSelector.desktop
./Engine/Source/Programs/UnrealVersionSelector/Private/Linux/Resources/com.epicgames.UnrealEngineEditor.desktop
./Engine/Source/Programs/UnrealVersionSelector/Private/Linux/Resources/com.epicgames.UnrealEngine.desktop
```

これはUnrealVersionSelector-Linux-Shippingを実行するとこれらのエントリーファイルが勝手に~/.local/shared/application以下にコピーされます. 

```bash
$ ./UnrealEngine-release/Engine/Binaries/Linux/UnrealVersionSelector-Linux-Shipping
```

この際ファイルパスは自動的に設定してくれますが, バイナリは自分で指定する必要があります.

```text
#!/usr/bin/env xdg-open

[Desktop Entry]
Version=1.0
Type=Application
Exec=/home/brainvader/Applications/UnrealEngine-release/Engine/Binaries/Linux/UE4Editor
Name=Unreal Engine Editor
Icon=ubinary
Terminal=false
StartupWMClass=UE4Editor
MimeType=application/uproject
Comment=Created By Unreal Engine
```

他のエントリファイル今のところ関係なさそうです.
